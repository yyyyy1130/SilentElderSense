# 需求对照表

本文档对照《计算机设计大赛》项目设计文档与实际开发情况。

---

## 一、项目目标

PDF要求聚焦3类高价值事件：

| 事件类型 | PDF要求 | 实际实现 | 对照结论 |
|---------|---------|---------|---------|
| **跌倒识别** | 识别突然倒地、缓慢倒地后长时间不起等异常情形 | YOLO11 ONNX模型检测fallen类别，连续5帧→MEDIUM，≥1秒→HIGH。无法区分突然/缓慢倒地 | ✅ 部分实现 |
| **长时间静止识别** | 识别室内人员长时间处于异常静止状态 | 30秒窗口内位移<阈值触发，阈值可配置；30秒→LOW，≥60秒→MEDIUM | ✅ 已实现 |
| **夜间异常活动识别** | 识别夜间反复徘徊、长时间滞留异常区域、起夜后异常停滞 | 仅检测"夜间时间段+长时间静止"组合，夜间静止→MEDIUM；未实现徘徊检测、异常区域滞留检测 | ⚠️ 部分实现 |

## 二、创新点

| PDF要求 | 实际实现 | 对照结论 |
|---------|---------|---------|
| **隐私优先的非接触式监测**：本地前景提取、人脸弱化、仅输出事件摘要 | Haar Cascade人脸检测 + GaussianBlur模糊处理；事件记录仅保存摘要信息（类型、风险、时间戳、持续时间），不保存原始视频；WebSocket实时传输已模糊的帧图像 | ✅ 已实现 |
| **规则与机器学习融合**：轻量视觉特征 + 规则引擎 + 传统ML分类器 | YOLO11深度学习检测模型 + 规则引擎风险分级判定（连续帧确认、时间阈值升级、夜间时段权重） | ✅ 已实现 |
| **完整系统化实现**：采集、识别、预警、日志、展示的完整软件平台 | 前端7个功能页面 + 后端完整API链路 + 数据库持久化 + ECharts图表 | ✅ 已实现 |

## 三、系统功能模块

| 功能模块 | PDF要求 | 实际实现 | 对照结论 |
|---------|---------|---------|---------|
| **视频输入与场景管理** | 支持摄像头/视频文件导入，房间/用户/时段配置 | 摄像头选择、视频文件上传；用户级检测参数配置（跌倒确认帧数、静止阈值、夜间时段） | ✅ 已实现 |
| **隐私预处理** | 前景分割、人体区域提取、匿名化显示 | Haar Cascade人脸检测 + GaussianBlur模糊处理 | ✅ 已实现 |
| **异常行为识别** | 跌倒、长时间静止、夜间异常活动 | YOLO11检测 + 规则引擎判定三类事件 | ✅ 已实现 |
| **风险评估与误报抑制** | 风险等级计算、滑动窗口/多帧投票/阈值校正 | 连续帧确认（5帧）、时间阈值升级、滑动窗口静止检测、1秒宽限期处理 | ✅ 已实现 |
| **应急响应** | 三级响应：本地提醒、管理端告警、紧急通知 | 三级响应框架（LOW→一级本地记录、MEDIUM→二级管理端、HIGH→三级紧急通知）；告警历史完整记录；第三方接口待对接 | ✅ 已实现 |
| **事件日志与可视化** | 事件记录、日/周统计、趋势图表、风险分布 | 事件表记录（类型、风险、时间、持续时间、状态）；事件类型分布饼图、风险等级分布饼图、处理状态柱状图；事件列表分页查询 | ✅ 已实现 |
| **用户与权限管理** | 管理员、家属、访客角色，不同角色看到不同内容 | 用户注册/登录、JWT认证；无角色权限区分 | ⚠️ 部分实现 |

## 四、技术路线

| 技术环节 | PDF要求 | 实际实现 | 对照结论 |
|---------|---------|---------|---------|
| **感知与预处理** | 背景建模、帧差法、轮廓检测、目标框跟踪 | YOLO11 ONNX模型检测人体姿态 + ByteTrack多目标追踪 + Haar Cascade人脸模糊 | ✅ 已实现 |
| **特征工程** | 目标框长宽比、质心位置变化、速度加速度、静止持续时间、夜间时间窗 | 目标框位置 + 质心位移计算 + 类别分类 + 静止持续时间 + 夜间时段判断 | ✅ 已实现核心特征 |
| **识别方法** | 规则 + 传统ML（随机森林/XGBoost/SVM） | YOLO11深度学习检测 + 规则引擎风险分级 | ✅ 已实现 |
| **多事件融合与风险分级** | 滑动融合、时间持续性、重复性形成风险分级 | 四级风险（NORMAL/LOW/MEDIUM/HIGH）；连续帧确认、时间阈值升级机制 | ✅ 已实现 |
| **服务端管理与可视化** | 事件汇总、风险统计、用户管理、告警展示、日志查询 | Quart后端API + SQLite数据库 + Vue3前端 + ECharts图表 | ✅ 已实现 |

## 六、差距说明

### 已实现功能

- 三类核心事件检测（跌倒、长时间静止、夜间异常）
- 四级风险分级机制
- 人脸模糊隐私保护
- 完整B/S架构系统
- 事件管理与可视化展示
- 三级告警响应框架

### 未完全实现功能

1. **跌倒检测**：无法区分"突然倒地"和"缓慢倒地"
2. **夜间异常检测**：未实现徘徊检测、异常区域滞留检测
3. **用户权限**：无角色权限区分（管理员/家属/访客）

### 待对接功能

- 短信/电话/微信等第三方告警通知接口
